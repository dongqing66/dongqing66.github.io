内网应用程序上线外网（内网穿透）

发表于2025-05-01|更新于2025-05-01
|浏览量:

当你关闭电脑或退出终端后，如果想重新测试程序，需要重新启动本地服务和 ngrok 的端口转发。以下是具体操作步骤：

📌 重新测试的完整流程

1. 启动你的本地程序

  • 确保你的项目在本地正常运行（比如Docker容器、Node.js服务等）。
  • 检查服务监听的端口（例如 3030 或 2283）：

2. 重新运行 ngrok 转发

  • 在终端执行（替换为你的实际端口）：
    • 注意：每次运行 ngrok 会生成新的随机域名（免费版），之前的域名失效。

3. 获取新域名并分享

  • 终端会显示新的转发地址（如 https://abcd-1234.ngrok-free.app），将此链接发给测试用户即可。

🔧 持久化方案（避免每次手动操作）

如果觉得每次重启都要重复操作太麻烦，可以用以下方法实现自动化：

1. 使用 tmux 或 screen 保持进程

2. 写一个启动脚本（一键重启）

  • 创建脚本 start_server.sh：
  • 赋予执行权限并运行：

3. 设置系统服务（长期运行）

  • Mac/Linux：用 launchd 或 systemd 将服务设为开机自启。
  • Windows：用任务计划程序设置开机启动。

🚨 注意事项

  1. 免费版Ngrok的限制：
    • 每次重启会更换域名，如需固定域名需付费升级。
    • 免费版有并发数和带宽限制。
  2. 长期解决方案建议：
    • 如果测试频率高，优先用 公网IP + DDNS（如之前教程）。
    • 或租用廉价VPS自建 Frp内网穿透（稳定且可控）。

💡 快速检查表

  • ✅ 本地服务是否运行？ lsof -i :端口号
  • ✅ Ngrok是否在线？ 访问 http://127.0.0.1:4040 查看状态页。
  • ✅ 新域名是否生成？ 检查终端输出的 Forwarding 地址。

是的！完全正确！

🔥 简化版操作流程（适用于“推倒重来”场景）：

  1. 确保 Docker 容器运行

  2. 测试本地访问

    ✅ 能正常访问 → 本地服务OK。
    ❌ 不能访问 → 检查Docker日志：docker logs 容器ID。

  3. 启动 ngrok 暴露服务

    • 终端会生成新的临时域名（如 https://abcd-1234.ngrok-free.app）。
    • 任何人访问这个域名 ≈ 访问你本地的 3030 端口。

  4. 分享链接给女友

    • 直接发她 https://xxxx.ngrok-free.app，无需其他配置！

⚠️ 注意事项

  1. 端口一致性：
    • docker run -p 3030:80 的第一个端口（3030）必须和 ngrok http 3030 的端口一致。
  2. 临时域名：
    • 每次重启 ngrok 会换域名，免费版无法固定。
  3. 关机和退出终端的影响：
    • 电脑关机或退出终端 → 需重新执行上述步骤（Docker + ngrok）。

🚀 如何更省事？

痛点 解决方案
每次重启要重跑命令 用 tmux 或脚本（见之前教程）
域名总变 升级Ngrok付费版 或 用DDNS+公网IP
担心安全 加 -auth "用户名:密码" 参数

📌 一句话总结

“推倒重来”时只需：

  1. Docker 跑起来（端口别错）。
  2. ngrok http 你的端口。
  3. 发新域名给女友。

完事儿！ 🌟
